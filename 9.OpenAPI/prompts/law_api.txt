# 역할

법제처 국가법령정보 공동활용 Open API를 이용하여 법령 검색과 판례 검색을 수행하는
Streamlit 기반 채팅형 웹 애플리케이션을 구축한다.
사용자가 자연어로 질문하면 AI(OpenAI GPT-4o-mini)가 의도를 분류하고,
적절한 API를 호출하여 결과를 채팅 메시지 형태로 보여주는 구조이다.

---

# 기술 스택

- **언어:** Python
- **프레임워크:** Streamlit
- **LLM:** OpenAI GPT-4o-mini (의도 분류 및 일반 질문 응답)
- **CSV 파싱:** pandas
- **HTTP 요청:** requests
- **환경 변수:** python-dotenv
- **외부 API:**
  - OpenAI API (GPT-4o-mini) — 사용자 의도 분류용
  - 법제처 Open API (law.go.kr/DRF) — 판례 목록/본문 조회용
- **데이터 파일:** 법령 목록 CSV 파일 (로컬 CSV 기반 법령 검색)

# 핵심 동작 흐름

## 1. 사용자 입력 → AI 의도 분류

사용자가 채팅창에 메시지를 입력하면 OpenAI GPT-4o-mini에 아래 시스템 프롬프트와 함께 전송하여 의도를 분류한다.

시스템 프롬프트:

당신은 국가법령정보 검색 시스템의 AI 어시스턴트입니다.
사용자의 입력을 분석하여 반드시 아래 JSON 형식으로만 응답하세요.

분류 기준:
"law": 특정 법령(법률·시행령·시행규칙·규정 등)을 검색하거나 법령 내용을 묻는 경우
"precedent": 판례·판결·사건을 검색하거나 판례 내용을 묻는 경우
"general": 법률 개념 설명, 법령/판례와 무관한 질문, 또는 검색 범위가 불명확한 경우

keyword 작성 규칙 (매우 중요):
- keyword에는 순수 검색 대상만 포함하세요.
- "판례", "법령", "법률", "사건", "검색", "조회" 같은 분류 용어는 keyword에 절대 포함하지 마세요.
- 법제처 API는 사건명(판례명) 기준으로 검색하므로, "판례" 같은 단어가 keyword에 포함되면 검색 결과가 0건이 됩니다.
- 예시: "음주운전 판례" → intent: "precedent", keyword: "음주운전"
- 예시: "근로기준법" → intent: "law", keyword: "근로기준법"
- 예시: "임대차보증금 판례" → intent: "precedent", keyword: "임대차보증금"
- 예시: "사기죄 관련 판례 검색" → intent: "precedent", keyword: "사기"

응답 JSON:
{
"intent": "law" | "precedent" | "general",
"keyword": "순수 검색 키워드만 (판례·법령·사건 등 분류 용어 제외)",
"response": "intent가 general일 때 사용자에게 보여줄 간결한 안내 텍스트"
}

OpenAI 호출 시 response_format={"type": "json_object"}, temperature=0, max_tokens=200을 설정한다.

---

## 1-1. 키워드 후처리 (안전장치)

GPT가 keyword에 노이즈 단어를 포함시키는 경우를 대비하여 반드시 후처리를 수행한다.

노이즈 단어 목록: "판례", "법령", "법률", "사건", "검색", "조회", "관련", "에 대한", "에 관한"

clean_keyword 함수로 위 단어들을 keyword에서 제거한 뒤 API에 전달한다.
제거 후 키워드가 빈 문자열이 되면 원본 키워드를 그대로 사용한다.

이 후처리는 intent가 "precedent"일 때만 적용한다.
(법령 검색은 CSV 법령명 매칭이므로 "근로기준법" 같은 전체 이름이 필요하다.)

---

## 2. 의도별 처리

### intent === "law" (법령 검색)

- data/법령검색목록_법령.csv 파일을 pandas로 로드한다 (@st.cache_data 사용).
- CSV 컬럼: 순번,법령MST,소관부처코드,소관부처명,법령ID,법령명,공포일자,공포번호,시행일자,법령구분코드,법령구분명,법령분야코드,법령분야명
- 첫 줄이 "총XXXX건"이면 스킵 (skiprows=1 또는 파싱 처리).
- 키워드가 법령명에 포함된 항목을 검색 (case-insensitive, 최대 15건).
- 각 법령의 URL은 법제처 한글 주소 규칙에 따라 생성:
  https://www.law.go.kr/법령/{법령명} (urllib.parse.quote 적용)

### intent === "precedent" (판례 검색)

- 법제처 Open API 호출: http://www.law.go.kr/DRF/lawSearch.do
- URL을 직접 문자열로 조립하고 query 파라미터는 urllib.parse.quote()로 명시적 인코딩한다.
  (requests의 params= 방식 대신 URL 직접 구성 — 한글 인코딩 호환성을 위해)
- 파라미터: OC={LAW_OC}&target=prec&type=JSON&query={urllib.parse.quote(keyword)}&page=1&display=10
- requests.get() 호출 시 반드시 User-Agent 헤더를 포함한다 (브라우저 UA).
  법제처 API는 User-Agent가 없거나 비표준이면 요청을 차단할 수 있다.
- resp.encoding = "utf-8" 을 명시적으로 설정한다.
- resp.json() 실패 시 응답 텍스트 일부를 에러 메시지에 포함하여 디버깅 가능하게 한다.

### intent === "general" (일반 질문)

- AI가 생성한 안내 텍스트를 채팅 메시지로 표시

# HTTP 요청 공통 사항 (중요)

법제처 API 호출 시 반드시 아래 사항을 적용한다:

1. **User-Agent 헤더 필수:** 브라우저 User-Agent 문자열을 헤더에 포함해야 한다.
   User-Agent가 없으면 법제처 서버가 요청을 거부할 수 있다.
2. **URL 직접 구성:** requests.get(url, params=...) 대신 URL 문자열을 직접 조립하고
   한글 파라미터는 urllib.parse.quote()로 인코딩한다.
3. **인코딩 명시:** resp.encoding = "utf-8" 을 설정한 후 resp.json()을 호출한다.
4. **JSON 파싱 에러 처리:** resp.json() 실패 시 응답 텍스트 앞부분을 에러 메시지에 포함한다.

## 채팅 영역

Streamlit의 st.chat_input()과 st.chat_message()를 사용한 채팅 인터페이스

- **사용자 메시지:** st.chat_message("user")
- **봇 메시지:** st.chat_message("assistant")
- **입력 플레이스홀더:** "법령명이나 판례 키워드를 자유롭게 입력하세요"
- **로딩:** st.spinner("검색 중...")

### 입력 메시지 즉시 표시 (중요)

사용자가 메시지를 입력하면 로딩 스피너가 도는 동안에도 사용자의 입력 메시지가 화면에 보여야 한다.

처리 순서:
1. 사용자 메시지를 st.session_state.messages에 추가
2. st.chat_message("user")로 사용자 메시지를 즉시 렌더링
3. st.chat_message("assistant") + st.spinner()로 로딩 상태 표시
4. 검색 실행 (의도 분류 → API 호출)
5. 결과를 st.session_state.messages에 추가
6. st.rerun()으로 전체 화면 갱신

사용자 메시지 추가와 검색 실행 로직은 별도 함수로 분리한다.
(예: execute_query 함수는 검색만 수행하고 사용자 메시지 추가는 main에서 처리)
