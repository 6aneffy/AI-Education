# 역할

너는 Python 기반 데이터 분석 시스템과 AI Agent를 구축하는 엔지니어다.

daejeon_price.db SQLite 물가 데이터베이스를 기반으로,
기본은 “대화형 자연어 물가 분석 Agent”로 동작하고,
사용자가 원할 때만 “동향 리포트 생성”을 수행하는
Streamlit 기반 물가 분석 시스템을 구축한다.

---

# 핵심 구조

이 시스템은 반드시 두 모드로 분리된다.

[기본 모드]
- 대화형 SQL Agent
- 질문 → 분석 → 시각화 → 해석 → 예측

[리포트 모드]
- 사용자가 버튼을 눌렀을 때만 실행
- 종합 분석 + 외부 원인 + 예측 + 그래프 묶기
- DOCX 보고서 생성

# UI 요구사항

## 1. 대화형 분석 영역 (기본)

- 채팅 UI 구성
- 입력창: “물가에 대해 질문하세요”
- 자동 분석 실행

Agent가 수행:
- 자연어 → SQL
- SQL 실행
- Pandas 분석
- 자동 그래프 생성
- 결과를 채팅 형태로 출력
- 그래프 화면 표시
- 외부 검색 (사이드바에서 선택 시)

---

## 2. 리포트 생성 영역 (선택)

분석 결과 후 안내 메시지 표시:
"위 분석 내용과 그래프를 기반으로 리포트를 생성해드릴까요?"

사용자가 확인하면:
- 현재 분석 대상 요약
- 누적 질의 기반 핵심 이슈 정리
- 월별 동향 분석
- OpenAI **GPT-5 nano** + Responses API **`web_search`** 기반 외부 원인 분석 (사이드바 설정에 따라, `OPENAI_API_KEY` 사용)
- 그래프 자동 수집
- DOCX 보고서 생성

생성 완료 시:
- 다운로드 버튼 제공
- output/reports에 저장

---

## 3. 사이드바 설정

- LLM 모델 선택 (gpt-4o-mini, gpt-5.3)
- 외부 검색 사용 체크박스 (OpenAI 웹 검색: GPT-5 nano + `web_search`, `OPENAI_API_KEY`)
- 대화 초기화 버튼 (모든 대화 기록 및 분석 데이터 초기화)

# 기능 요구사항

## A. DB 분석 Agent

- sqlite3 / SQLAlchemy 연결
- LLM 기반 자연어 → SQL
- 쿼리 검증
- 다중 SQL 실행 가능
- Pandas 분석
- 품목 추출 로직 (데이터프레임, SQL 쿼리, 질문에서 추출)
  - 데이터프레임의 item 컬럼 우선 사용
  - SQL 쿼리의 WHERE 조건에서 추출
  - LLM을 사용한 질문 분석 (DB 품목 목록 참고)
  - 질문 텍스트에서 직접 매칭

## B. 시각화 엔진

그래프 저장:
output/charts/

## C. OpenAI 웹 검색 연동 (GPT-5 nano, Responses API)

- 사이드바에서 외부 검색 허용 여부 설정 가능
- 기본 모드에서도 사용 가능 (선택적)
- 리포트 모드에서도 사용 가능 (선택적)
- 품목 추출 후 자동으로 외부 원인 분석 수행
- 품목 + 기간 기반 검색 질의를 `responses.create` 의 `input` 으로 전달
- 원인 요약 생성 (모델 출력은 텍스트·인용 블록 형태일 수 있음 → 앱 규칙에 맞게 평문 처리)
- 분석 결과에 외부 원인 정보 포함
- 품목을 찾을 수 없으면 적절한 메시지 표시
- 구현은 **`openai` SDK** 권장: `OpenAI().responses.create(model="gpt-5-nano", tools=[{"type": "web_search"}], input=..., previous_response_id=...)` . **`OPENAI_API_KEY`** 만 사용 (Perplexity 키 없음).
- **알려진 오류:** `langchain_openai.ChatOpenAI.bind_tools([{"type": "web_search"}])` 는 LangChain 버전에 따라 **`ValueError: Unsupported function`** 이 발생한다. 웹 검색 경로에서는 `bind_tools` 로 `web_search` 를 넘기지 말 것.
- 401·할당량·연결 오류 시 사용자에게 안내하고 로그에 기록한다.

---

## E. DOCX 생성기 (리포트 모드 전용)

python-docx 사용

구성:
- 표지
- 요약
- 데이터 분석
- 그래프 섹션
- 외부 원인 분석
- 종합 결론

그래프 섹션:
- 리포트 생성 시 차트를 PNG로 저장한 뒤 DOCX에 HTML 경로가 아닌 이미지로 삽입한다.
- 보고서를 열었을 때 그래프가 시각화된 그림으로 바로 보이도록 한다.
- 기본 방식은 Matplotlib/Seaborn으로 PNG를 생성하고 doc.add_picture()로 삽입한다.
- Plotly는 화면용(인터랙티브)으로만 사용 가능하며, 보고서용 이미지는 정적 PNG로 다시 생성한다.
- kaleido는 필수 의존성이 아니며, 사용하지 않아도 리포트 생성이 가능해야 한다.

# 대화 응답 형식 (기본 모드)

- 해석 텍스트
- 핵심 수치 bullet
- 그래프 표시
- 외부 검색 결과 포함 (사이드바에서 허용 시)

외부 검색 결과 표현 규칙:
- 응답은 DOCX 삽입을 고려해 항상 일반 텍스트 중심으로 작성한다.
- 마크다운 문법(예: #, ##, **, *, _, `, >, 코드블록, 링크 문법)을 출력하지 않는다.
- 목록은 불렛 리스트(예: "- 원인 1", "- 원인 2") 형태의 순수 텍스트만 사용한다.
- 표가 필요하면 텍스트 표기(열 이름: 값)로 풀어쓴다.
- 링크가 필요한 경우 URL 원문만 한 줄 텍스트로 기재한다.
- 요약은 짧은 문단 + 불렛 리스트 조합으로 작성한다.

❌ DOCX 생성 금지 (리포트 모드에서만 생성)
